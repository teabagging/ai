Track Processed Characters: Use a set to keep track of characters that have already been processed (i.e., their second occurrence has been checked).3. Check for Second Occurrence: As we iterate through the string, if a character is found that has been seen before (exists in the dictionary) and hasn't been processed, we check if this second occurrence has the smallest index encountered so far.4. Update Result: If the current second occurrence index is smaller than the previously found smallest index, update the result to this character and adjust the smallest index accordingly.This approach ensures that each character is processed in linear time, making the solution efficient with a time complexity of O(n) and a space complexity of O(1) (since the dictionaries and sets used will have at most 26 entries for lowercase letters).
