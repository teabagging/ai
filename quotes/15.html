The solution should be efficient in terms of both time and space complexity. ApproachThe optimal approach to solve this problem is using Floyd's Tortoise and Hare algorithm. This algorithm helps in detecting cycles in a sequence, which is useful here because the presence of a duplicate integer creates a cycle in the array when viewed as a linked list where each value points to the next index. The algorithm consists of two phases:1. Finding the meeting point: Use two pointers (tortoise and hare) where the tortoise moves one step at a time and the hare moves two steps at a time. They will eventually meet inside the cycle.2. Finding the entrance to the cycle: Reset one pointer to the start and move both pointers one step at a time. The point where they meet again is the start of the cycle, which corresponds to the duplicate value. Solution Code```pythondef find_dup(arr): # Phase 1: Find the intersection point of the two runners. tortoise = arr[0] hare = arr[0] while True: tortoise = arr[tortoise] hare = arr[arr[hare]] if tortoise == hare: break # Phase 2: Find the entrance to the cycle (the duplicate number). tortoise = arr[0] while tortoise != hare: tortoise = arr[tortoise] hare = arr[hare] return hare``` Explanation1. Phase 1: The tortoise and hare start at the first element of the array. The tortoise moves one step (i.e., `tortoise = arr[tortoise]`) while the hare moves two steps (i.e., `hare = arr[arr[hare]]`). They continue moving until they meet, which confirms the presence of a cycle.2. Phase 2: Once a meeting point is found, reset the tortoise to the start of the array. Both the tortoise and hare then move one step at a time. The point where they meet again is the start of the cycle, which is the duplicate value.This approach efficiently finds the duplicate in O(n) time complexity with O(1) space complexity, making it optimal for large arrays.
